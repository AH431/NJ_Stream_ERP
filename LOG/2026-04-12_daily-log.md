#### 1-4. SyncProvider（`lib/providers/sync_provider.dart`）

完整實作，對齊 Sync Contract v1.6：

- **Proactive Refresh**（Issue #3 AC）：過期前 5 分鐘自動排程刷新，`_isRefreshing` flag 防並發
- **被動 Refresh 備援**：Dio interceptor 攔截 401，自動 retry 原始請求
- **`pushPendingOperations()`**：
  - 依 `id` 升序讀取 `pending` 佇列，每批上限 50 筆（Sync Contract §5）
  - 推送前標記 `syncing`，防止重複推送
  - 逐筆處理 server result，對應 §6 五種錯誤碼策略
- **`_handleError()`** 對齊 §6：
  - `FORBIDDEN_OPERATION / PERMISSION_DENIED / VALIDATION_ERROR` → Force Overwrite
  - `INSUFFICIENT_STOCK / DATA_CONFLICT` → 標記 failed
  - 未知錯誤 → incrementRetry + 回退 pending
- **效能優化**：
  - `_jwtPayload` 快取：`userId` / `role` getter 不再每次重新 decode JWT
  - `_markBatch` 非 retry 路徑改為單條 `WHERE id IN (...)` 批次 UPDATE
  - `_countPending` 改為 `SELECT COUNT(*)`，不載入 row 資料
- **`login()`**：呼叫 `/api/v1/auth/login`，存雙 token
- **`logout()`**：清除 timer + token + payload + SecureStorage
