### 過程中發現並修正的問題

#### Bug 1：空列表 RefreshIndicator 無法觸發

**問題**：`inventory_list_screen.dart` 與 `sales_order_list_screen.dart` 在列表空白時
顯示 `Center` 文字，`RefreshIndicator` 不在畫面上，下拉手勢無效。

**修正**：
- 將 `RefreshIndicator` 移到 empty check 外層
- 空狀態改用 `ListView`（含 `AlwaysScrollableScrollPhysics`）包裝，確保下拉永遠有效
- 修改檔案：`inventory_list_screen.dart`、`sales_order_list_screen.dart`

#### Bug 3：sales_order:create FORBIDDEN_OPERATION 後本地記錄未回滾

**問題**：報價轉訂單被伺服器拒絕（FORBIDDEN_OPERATION，報價已被轉換）時，
本地 `sales_orders` 表中樂觀寫入的記錄未清除，導致訂單列表出現無法操作的
「待處理」殘留訂單（id < 0，無取消按鈕，因 `canCancel` 要求 `!isOffline`）。

**重現路徑**：離線建立報價 → 同步 → 誤觸已轉換報價的「轉訂單」→ FORBIDDEN_OPERATION → 殘留。

**修正方向**：`_handleError` 中對 `sales_order:create` 的 FORBIDDEN_OPERATION
應額外呼叫 `db.deleteSalesOrder(localId)`（軟刪除，設 deletedAt），
讓 `watchActiveSalesOrders()` 自動過濾掉殘留記錄。

---

#### Bug 2：ShipOrderDialog 出貨前需手動 Pull

**問題**：reserve Push 後，本地庫存快照未自動更新，ShipOrderDialog 讀到 reserved=0
而非 reserved=3，顯示 ⚠️ 警示「尚未預留，服務端將拒絕」。

**暫行方案**：出貨前手動 Pull-to-refresh（Tab 5）更新本地快照。

**待辦**：在 `_shipOrder()` 或 ShipOrderDialog 開啟前自動呼叫 `pullData()`，
或加入提示「建議出貨前先同步以取得最新庫存狀態」。

## UI 改善（同日）

### FAB 圖示統一

**問題**：客戶 FAB 使用 `person_add_outlined`（單一 24dp 設計圖示），產品和報價 FAB
只有 `Icons.add`，視覺不一致。

**過程**：嘗試 Stack 疊加與 Row 並排兩種方案，均因兩個獨立圖示的線重無法對齊
單一設計圖示而放棄，最終改用內建含 + 的 Material 圖示。

**修正**：

| Tab | 舊圖示 | 新圖示 |
|-----|--------|--------|
| 客戶 | `person_add_outlined` | 不變 |
| 產品 | `Icons.add` | `library_add_outlined` |
| 報價 | `Icons.add` | `note_add_outlined` |
| 庫存入庫 | `add_box_outlined` | 不變 |

**設計決策**：FAB 一律使用 Material 單一設計圖示，不拼接兩個圖示，確保線重一致。

### UI 設計規範確立

使用者回饋確立以下原則，適用所有後續 feature screen 實作：

- **標籤 / 狀態**：純文字 + 色彩 + 圖示（`Row(Icon + Text)`），無外框無背景
- **按鈕**：`TextButton.icon` / `OutlinedButton` / `FilledButton`，有外框或填色
- **整體**：專注 visual hierarchy and consistency
