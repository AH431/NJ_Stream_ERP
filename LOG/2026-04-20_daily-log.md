## 二、CSV 匯入問題根本原因診斷

### 問題描述

2026-04-20 測試時，CSV 匯入完全無法運作：後端 log 完全收不到任何 `/api/v1/admin/import` 請求，表示上傳從未被觸發。

對比 **2026-04-17 測試（通過）** 與今日（失敗），確認問題始於 **2026-04-18 commit `b8379b6`**。

### 根本原因

`b8379b6` 新增 `open_filex: ^4.4.0`，為此在 `AndroidManifest.xml` 註冊了 `FileProvider`：

```xml
<provider
    android:name="androidx.core.content.FileProvider"
    android:authorities="${applicationId}.fileProvider"
    ...>
    <meta-data android:resource="@xml/file_paths" />
</provider>
```

同時新增的 `file_paths.xml` **只涵蓋 cache 路徑**：

```xml
<cache-path name="cache" path="." />
<external-cache-path name="external_cache" path="." />
```

**Android 8.0（Sony XA1）上的 `file_picker` 套件**在處理外部儲存（如 `/sdcard/Download/`）的選取結果時，會呼叫 `FileProvider.getUriForFile()` 建立可分享 URI。由於 `file_paths.xml` 未涵蓋 `/sdcard/Download/` 路徑，`FileProvider` 拋出 `IllegalArgumentException: Failed to find configured root`，`file_picker` 捕捉後靜默回傳 `null`，導致上傳永遠不被觸發。

### 排查過程

1. 對比 commit 歷史：`bc5c62a`（4/16 匯入功能初版）→ `301ec7d`（4/17 測試通過）→ `b8379b6`（4/18 加入 open_filex，問題開始）
2. 確認後端日誌：登入 ✓、pull ✓、cleanup ✓，但匯入請求為零 → 問題在 client 端
3. 確認資料庫與 API 權限無問題（`postgres` superuser + admin role 全域可寫）
4. 確認是 `file_paths.xml` 路徑覆蓋不足（非 Activity recreation 或 largeHeap 問題）
5. 4/18 phone-test-guide 本身即記載「FileProvider crash | Android 版本問題」為已知潛在風險

## 三、修正內容

### 3-1 `file_paths.xml` — 擴充 FileProvider 路徑覆蓋

**檔案**：`packages/frontend/android/app/src/main/res/xml/file_paths.xml`

```xml
<!-- 修正前 -->
<cache-path name="cache" path="." />
<external-cache-path name="external_cache" path="." />

<!-- 修正後（新增三條） -->
<files-path name="files" path="." />
<external-path name="external" path="." />          ← 涵蓋 /sdcard/Download/
<external-files-path name="external_files" path="." />
```

讓 `file_picker` 對 `/sdcard/` 下任意路徑建立 FileProvider URI 不再失敗。

### 3-2 `AndroidManifest.xml` — 加入 `largeHeap`

**檔案**：`packages/frontend/android/app/src/main/AndroidManifest.xml`

```xml
<application android:largeHeap="true" ...>
```

減少 Sony XA1 低記憶體環境下，開啟外部 Activity 時 App 被系統回收的機率。

### 3-3 `import_screen.dart` — 完全重新設計為資料夾掃描流程

**檔案**：`packages/frontend/lib/features/settings/import_screen.dart`

**背景**：即使修正 `file_paths.xml`，FilePicker 在 Sony XA1 Android 8.0 上仍不穩定（Activity recreation 導致 result 丟失）。為根除問題，改為 `dart:io` 直讀本機路徑，完全繞過系統 File Picker。

**舊設計**：FilePicker 選檔 → 上傳

**新設計**（`dart:io` + 資料夾掃描）：

| 步驟 | 行為 |
|------|------|
| 開啟畫面 | 自動載入 `getExternalStorageDirectory()/test_csv/` 並掃描 |
| 切換類型 | 自動重新篩選，只顯示檔名含「product / customer / inventory」的 CSV |
| 點選檔案 | Radio 選中 + 顯示前 8 行內容預覽（藍色框） |
| 點「確認匯入」 | `dart:io File.readAsBytes()` 讀檔 → `uploadImportCsv()` 上傳 |
| 找不到資料夾 | Orange hint 顯示 `adb push` 指令 |

**關鍵技術決策**：
- 使用 `getExternalStorageDirectory()`（`path_provider`，已有相依）取得 App 自身 external storage，**無需 `READ_EXTERNAL_STORAGE` 權限**
- `FileSystemException` 完整包在 try-catch 內，OS error message 顯示於 dialog
- 移除 FilePicker 按鈕（`file_picker` 套件 import 保留，日後其他設備可恢復）

**測試檔案推送路徑（adb）**：
```bash
adb push LOG/test_csv/ /sdcard/Android/data/com.example.nj_stream_erp/files/test_csv/
```

**實測結果**：dart:io 路徑手動輸入方式可成功匯入（2026-04-20 下午測試通過）。

### 3-4 `app_strings.dart` / `main.dart` — 語言設定持久化

**問題**：雙語 UI 以 `_isEnglish` 記憶體狀態保存，Sony XA1 Activity recreation 後 App 重啟，`AppStrings()` 重新建立，語言強制回到中文。

**修正**：
- `AppStrings` 建構子注入 `FlutterSecureStorage`（已有相依）
- `init()` 方法從 secure storage 讀取 `app_language` 鍵
- `setEnglish(bool)` 改為 `async`，每次切換同步寫入 storage
- `main()` 改為 `async`，`await appStrings.init()` 後再 `runApp()`，確保 UI 初始化時語言已還原
- `ChangeNotifierProvider` 改為 `.value(value: appStrings)` 傳入預先初始化的實例

**影響**：App 任何重啟（含 Activity recreation）後，語言設定均自動還原。
