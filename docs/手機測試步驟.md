### 0-2.5 確認手機測試 Demo 資料已載入

> ⚠️ **這步很重要。** 只要你有重建 Docker、切換 `.env`、換 DB session、換資料庫容器，**都要重新確認一次**。  
> 若略過這步，手機可以成功登入，但 Dashboard / Analytics 會全部空白，看起來像前端壞掉，其實是後端資料庫沒有 Demo seed。

**固定操作原則：**

- `npm run dev` 只負責**啟動後端**
- `npm run seed:phone-demo` 只負責**補手機測試用英文 Demo 資料**
- 兩者是不同步驟，不要混在一起理解

**建議固定執行順序：**

```powershell
cd c:/Projects/NJ_Stream_ERP/packages/backend

# 1. 若測試帳號尚未建立，先補這步
node node_modules/tsx/dist/cli.mjs scripts/seed-test-user.ts

# 2. 常駐英文版手機測試 Demo seed（04/26 驗證版的固定入口）
npm run seed:phone-demo

# 3. 啟動後端
npm run dev
```

**固定入口檔**：`packages/backend/scripts/seed-phone-demo-en.ts`  
**資料來源**：沿用 2026-04-26 驗證成功的 `seed-demo-data.ts`  
**冪等性**：若 `TUBE-A001` 已存在，腳本會安全跳過，不會重複插入。

**什麼時候一定要重跑 `npm run seed:phone-demo`：**

- 重建 Docker / PostgreSQL 容器後
- 修改 `packages/backend/.env` 或 `DATABASE_URL` 後
- 切換 DB session / 不確定目前是不是同一個資料庫時
- 手機可以登入，但 Dashboard / Analytics 顯示空白時

```bash

## 已知問題 / 備案

| 問題 | 原因 | 解法 |
|------|------|------|
| PDF 點按無反應 | 手機無 PDF 閱讀器 | 安裝 Google PDF Viewer |
| PDF 中文顯示方塊 | 後端缺字型 | 確認 `assets/fonts/NotoSansTC-VariableFont_wght.ttf` 存在 |
| 寄信後未收到 | Gmail 應用程式密碼失效 | 重新產生並更新 `.env` |
| CSV 掃描不到檔案 | adb push 路徑錯誤 | 確認推送到 `…/files/test_csv/`（非 Download/） |
| App Activity recreation 後語言重置 | 已修正（4/20） | 確認 `AppStrings.init()` 在 `main()` 的 `await` 順序正確 |
| B-8 heatmap 500 Error | ~~Drizzle ANY() 參數問題~~ | ✅ 已修復（4/26）：改用 `sql.join` + `IN` 語法 |
| FCM 推播無法測試 | Firebase Console 未設定 | 需建立 Android App、下載 google-services.json、產生 Service Account JSON |
| DB migration IPv6 連線失敗 | Node.js 24 優先解析 IPv6 `::1`，Docker 僅綁定 IPv4 | 執行 migrate 時加 `DATABASE_URL="...@127.0.0.1:5432/..."` 明確指定 |
