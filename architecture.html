Key Conventions
  • Use Decimal type for all monetary display — never double or float
  • All mutations go through SyncProvider → PendingOperations table → batch push
  • Overlay buttons must be the last child in a Stack (scroll intercept bug)
  • FilledButton global theme: use Size(64, 52) not infinity width (crashes inside Row)
  • Negative local IDs until server assigns real IDs (offline ID remap protocol)
  • Use Iso8601DateTimeConverter in Drift tables for timestamp columns

Sync Protocol v1.6 (Offline-First)
Spec: docs/NJ_Stream_ERP MVP 同步協定規格 v1.6.md · ADRs: 001 (LWW), 002 (ID remap)
Push Flow
1
Mutation happens offline → written to PendingOperations (status: pending)
2
Network recovered → SyncProvider.performSync() batches ≤50 ops by created_at ASC
3
POST /api/v1/sync/push with JWT Bearer
4
Backend: idempotency check (operationId in processed_operations) → process in DB transaction
5
Partial success: { succeeded: [id…], failed: [{ id, code, message }…] }
6
Frontend: update local state, remap negative local IDs to real server IDs, mark ops succeeded/failed
Conflict Resolution (LWW)
If client.updatedAt < server.updatedAt, server rejects with DATA_CONFLICT. Client then force-pulls to get the latest server version before retrying.
Local ID Remap (Offline ID Chain)
  • New offline entities get negative integer IDs locally (e.g. -1, -2, -3…)
  • After parent entity (e.g. customer) create succeeds, server returns the real positive ID
  • All batch-internal FKs referencing the old negative ID are updated before the next op in the same batch
  • ID mapping stored in RemapDao in Flutter

This tab is a quick-start for the next agent adding a new feature. Follow these patterns exactly to stay consistent with the existing codebase.
Adding a Backend Route
  1. Create src/routes/<feature>.route.ts
  2. Register in src/app.ts with prefix /api/v1/<feature>
  3. Add role guard in auth.plugin.ts or inline preHandler
  4. If mutating data: add entity type to /sync/push handler OR use direct REST endpoint
  5. Apply rate limit if user-facing or CPU-heavy
  6. Do NOT add side effects to GET /health
Adding a Database Table
  1. Add table definition to packages/backend/src/db/schema.ts
  2. Run: npm run db:generate && npm run db:migrate
  3. Add deletedAt column for master data (soft-delete pattern)
  4. Use numeric(12,2) for ALL monetary columns
  5. Use timestamp with timezone for all time columns
  6. Add corresponding Drift table in packages/frontend/lib/database/
Adding a Flutter Screen
  1. Create screen in packages/frontend/lib/screens/<feature>/
  2. Create Provider in lib/providers/<feature>_provider.dart if needed
  3. Register Provider in MultiProvider in lib/main.dart
  4. Register route in MaterialApp routes or use Navigator.push
  5. If screen modifies data: add PendingOperation via SyncProvider
  6. Use Decimal type for all monetary display — never double
  7. Overlay buttons must be the last child in a Stack
Adding Sync Support for New Entity
  1. Add entity type string to supported list in /sync/push handler
  2. Add case to switch statement in sync route processing logic
  3. Add idempotency check against processed_operations
  4. Add local Drift table + DAO + PendingOperation logic in Flutter
  5. Document in docs/NJ_Stream_ERP MVP 同步協定規格 v1.6.md
Critical Conventions
ConventionRule
Monetary valuesnumeric(12,2) in DB → string in JSON → Decimal in Dart. Never float/double.
Soft-deleteSet deletedAt = now(). All list queries must WHERE deletedAt IS NULL.
IdempotencyUUID v4 operationId generated client-side. Backend checks processed_operations before processing.
Timestampstimestamp with timezone → ISO-8601 JSON → Iso8601DateTimeConverter in Drift
Role checkAdd to auth.plugin.ts preHandler. sales/warehouse/admin defined in users.role enum.
Rate limitingRegister with Fastify rate-limit plugin in src/app.ts or per-route config.
Anomaly severitycritical triggers FCM push. high and medium only show in bell.
Flutter StackOverlay buttons must be last child in Stack or ScrollView intercepts taps.
FilledButton widthUse Size(64, 52) in global theme — never infinity width (crashes inside Row).
Vector embeddingsUse mxbai-embed-large — bge-m3 produces NaN on Markdown text.
